## ¿Qué es Git?

Git es una herramienta de control de versiones que me permite registrar cada cambio que realizo en mi código. Funciona como una máquina del tiempo donde puedo guardar estados específicos de mi proyecto y regresar a ellos si cometo un error, facilitando mucho el proceso de desarrollo.

## Por qué necesitamos Git

Sin Git, trabajar en proyectos grandes o en equipo sería muy difícil porque los archivos se duplicarían o se borrarían cambios por accidente. Git resuelve esto permitiendo que varios programadores trabajen en el mismo código de forma organizada, manteniendo un historial claro de quién hizo qué.

## Los 3 conceptos clave de Git

1. **Repositorio:** Es la carpeta donde se guarda todo mi proyecto y su historial de cambios gestionado por Git.
2. **Commit:** Es un punto de guardado oficial en el historial, como una captura de pantalla de cómo estaba mi código en ese momento.
3. **Rama:** Es una línea de tiempo alterna que me permite probar cosas nuevas sin dañar la versión principal que ya funciona.

## Los comandos que aprendí

- `git init`: Para iniciar un repositorio nuevo.
- `git status`: Para ver el estado actual de mis archivos.
- `git add`: Para marcar los archivos que quiero incluir en mi próximo guardado.
- `git commit -m "mensaje"`: Para guardar los cambios con una descripción.
- `git push`: Para subir mi trabajo de mi computadora a GitHub.
- `git pull`: Para bajar los cambios más recientes desde GitHub a mi máquina.
